fix_unused_class_weights

<DESCRIPTION>
The current program computes class weights to handle the imbalanced dataset but fails to use them in the loss function. The `w` tensor is calculated in `train_classifier()` but ignored. This is a critical bug, especially since class 2 (severe PD) has F1=0.00, indicating it's being ignored.

We replace FocalLoss with nn.CrossEntropyLoss(weight=w) to directly apply the computed inverse-frequency weighting. This forces the model to pay more attention to underrepresented severe PD cases, which is crucial for clinical impact. The change is minimal but addresses the most glaring issue in the current pipeline.
</DESCRIPTION>
