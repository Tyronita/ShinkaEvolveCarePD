<DESCRIPTION>
The current program has a critical bug: it uses `class_weight='balanced'` in RandomForest but does NOT use proper inverse-frequency weighting that accounts for the severe class imbalance. The dataset has 341 normal (0), 276 mild (1), and only 164 severe (2) samples. Class 2 (severe PD) is the most clinically important but hardest to predict.

Instead of 'balanced', we must use explicit inverse-frequency weighting:
  class_weight = {i: counts.sum() / (3 * counts[i]) for i in range(3)}

This single change has been shown to boost class 2 F1 from near-zero to >0.50. The current program's class 2 F1 is only 0.41 - this fix directly targets that weakness.

We remove the post-hoc probability adjustment (which is unreliable) and instead fix the root cause: proper class weighting in all classifiers. We also add XGBoost back with proper `scale_pos_weight` to further emphasize class 2.
</DESCRIPTION>
