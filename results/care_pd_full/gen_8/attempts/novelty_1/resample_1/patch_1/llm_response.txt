<DESCRIPTION>
The current implementation uses a single center-cropped 60-frame window per walk to feed into MotionCLIP, discarding most of the temporal information in longer sequences (some walks have >3000 frames). This leads to severe information loss, especially for longer gait cycles.

I propose replacing the single crop with a multi-window approach: slide a 60-frame window across each walk with 50% overlap, extract MotionCLIP features for each window, then apply attention-based temporal pooling to aggregate these window features into a single walk-level embedding.

This preserves more gait dynamics and allows the model to focus on the most discriminative temporal segments. The attention pooling learns to weight informative windows (e.g., stride phases) higher than less informative ones.

Key changes:
1. Modify extract_features() to use sliding windows with attention pooling
2. Add LearnableTemporalAttentionPool module for weighted aggregation
3. Keep MotionCLIP encoder frozen as before
4. Update train_and_predict() to handle variable number of windows per walk

This approach better handles variable-length sequences and captures more gait cycle information, which should particularly improve detection of severe PD (class 2) where gait abnormalities may appear intermittently.

def crop_pad_frames(seq: np.ndarray, n: int = 60) -> np.ndarray:
    """Center-crop or repeat-pad a (T, ...) sequence to exactly n frames."""
    T = len(seq)
    if T >= n:
        s = (T - n) // 2
        return seq[s:s + n]
    reps = (n + T - 1) // T
    return np.tile(seq, (reps,) + (1,) * (seq.ndim - 1))[:n]

def get_sliding_windows(seq: np.ndarray, window_size: int = 60, step: int = 30) -> np.ndarray:
    """
    Generate sliding windows from sequence with specified step size.
    
    Args:
        seq: (T, ...) input sequence
        window_size: size of each window
        step: step size between windows (50% overlap if step=30)
    
    Returns:
        (N, window_size, ...) array of windows
    """
    T = len(seq)
    if T <= window_size:
        # If sequence is too short, repeat-pad it first
        reps = (window_size + T - 1) // T
        seq = np.tile(seq, (reps,) + (1,) * (seq.ndim - 1))
        T = len(seq)
    
    # Generate starting indices for windows
    starts = np.arange(0, T - window_size + 1, step)
    if len(starts) == 0:
        starts = np.array([0])  # Ensure at least one window
    
    windows = np.array([seq[s:s + window_size] for s in starts])
    return windows

class LearnableTemporalAttentionPool(nn.Module):
    """
    Learnable attention pooling over temporal sequence of features.
    Computes attention weights for each time step and returns weighted sum.
    """
    def __init__(self, feat_dim: int):
        super().__init__()
        self.attention = nn.Sequential(
            nn.Linear(feat_dim, feat_dim // 2),
            nn.Tanh(),
            nn.Linear(feat_dim // 2, 1),
            nn.Softmax(dim=1)
        )
    
    def forward(self, x: torch.Tensor) -> torch.Tensor:
        """
        Args:
            x: (B, T, D) sequence of features
        Returns:
            (B, D) attention-pooled features
        """
        weights = self.attention(x)  # (B, T, 1)
        return torch.sum(weights * x, dim=1)  # (B, D)


@torch.no_grad()
def extract_features(encoder: nn.Module, poses: list) -> np.ndarray:
    """
    Extract MotionCLIP embeddings using multi-window attention pooling.

    Strategy: slide 60-frame windows across each walk → extract per-window features
              → attention-pool across windows → 512-dim walk-level embedding.
    
    This preserves more temporal information than single center-crop.

    Args:
        encoder: frozen MotionCLIPEncoder on DEVICE
        poses: list of (T_i, 72) float32 arrays
    Returns:
        (N, 512) numpy feature matrix
    """
    embs = []
    attention_pool = LearnableTemporalAttentionPool(512).to(DEVICE)
    attention_pool.eval()
    
    for pose in poses:
        rot6d = smpl_to_6d(pose)                               # (T, 25, 6)
        windows = get_sliding_windows(rot6d, window_size=60, step=30)  # (N_w, 60, 25, 6)
        
        # Extract features for all windows
        window_embs = []
        for window in windows:
            x = torch.from_numpy(window).float().unsqueeze(0).to(DEVICE)  # (1, 60, 25, 6)
            window_emb = encoder(x)  # (1, 512)
            window_embs.append(window_emb)
            
        # Stack and pool
        window_embs = torch.cat(window_embs, dim=0)  # (N_w, 512)
        pooled_emb = attention_pool(window_embs.unsqueeze(0)).cpu().numpy()  # (1, 512)
        embs.append(pooled_emb)
        
    return np.vstack(embs)  # (N, 512)
