<DESCRIPTION>
The current program calibrates only the Random Forest (RF) classifier using `CalibratedClassifierCV`, but leaves the Gradient Boosting (GB) classifier uncalibrated. This creates a bias in the soft-voting ensemble, as GB tends to produce overconfident and miscalibrated probabilities — especially underestimating the severe PD class (class 2). Since the ensemble averages raw probabilities from both models, an uncalibrated GB can pull predictions toward normal/mild classes, directly harming macro-F1.

We should apply the same isotonic calibration to GB as is used for RF. This ensures both ensemble members contribute well-calibrated probabilities, particularly improving the representation of the underrepresented severe class. This change targets the root cause of the current 0.45 F1 on class 2, without requiring new features or architecture changes.

This edit wraps the GB pipeline in `CalibratedClassifierCV` with `method='isotonic'` and `cv=3`, matching the RF calibration strategy. It follows the winning strategy of using calibrated ensembles for imbalanced clinical classification.
</DESCRIPTION>
